CSS 101

2. Syntaxe

Hlavní strana > Web > Front⁠-⁠End > CSS > CSS 101 > Lekce 2

Syntaxe je řada pravidel, kterými se musí kód řídit. Každý programovací jazyk má nějakou, od assemberu Intel 4004 přes Python až po HTML a CSS. A právě o syntaxi CSS si v této lekci řekneme.

Syntaxe CSS je velmi jednoduchá - nic se nemusí dělat v žádném daném pořadí, nemusí se využívat tabulátory (i když je dobrou praktikou je kvůli přehlednosti používat) a podobně. V CSS se navíc používá jedna struktura - pravidla.

Pravidla

Pravidla jsou základními bloky CSS kódu, které většinou odkazují na nějaký specifický element, který posléze stylují.

        p {
            color: red;
        }

Skládají z několika částí:

  • Selektor: Specifikuje, na který element pravidlo odkazuje. Nachází se vždy na začátku pravidla. Selektorem v příkladě výše je p, což znamená, že pravidlo odkazuje na všechny odstavce. Více si o nich řekneme v následující lekci.
  • Složené závorky {}: Obklopují deklarace. Následují hned po selektoru.
  • Deklarace: Určují styl pro elementy, na které selektor odkazuje. Skládají se ze dvou částí - vlastnosti a její hodnoty, přičemž oba prvky jsou odděleny dvojtečkou :. Každá deklarace je poté zakončena středníkem ;. V příkladě výše se nachází jen jedna deklarace, kterou tvoří vlastnost color s hodnotou "red". To znamená, že barva elementu, na který selektor odkazuje, bude červená. Deklarací se v každém pravidle může nacházet neomezeně. Na konci toho posledního se sice středník nemusí, ale je to doporučeno.

Pravidlo výše tedy znamená, že všechny odstavce budou mít červenou barvu.


 

Řádkování

V CSS nezáleží na tom, co se nachází na jakém řádku. Všechny příklady níže jsou tedy validní:

        p {
            color: red;
        }
        p {color: red;}
        p
        {
            color:
            red;
        }

Pro přehlednost je dobrou praktikou psát selektor a počáteční složenou závorku na jednom řádku, každou deklaraci na jiný následující řádek, a uzavírající složenou závorku na řádek po poslední deklaraci.

Priorita pravidel

V CSS platí, že vyšší prioritu má pravidlo, které se nachází v kódu níže. Jazyk je totiž prohlížeči interpretován řádek po řádku. Dejme tomu, že existuje pravidlo, které změní barvu všech odstavců na červenou. Mělo by fungovat normálně.

        p {
            color: red;
        }

Pokud se ale kdekoliv pod to pravidlo napíše jiné, které změní barvu všech odstavců na modrou, tak prohlížeč bude ignorovat to první a bude se řídit tím níže položeným.

        p {
            color: red;
        }

        p {
            color: blue;
        }

Komentáře

Stejně jako prakticky všechny programovací jazyky umožňuje i CSS používat komentáře. Ty jsou vlastně části kódu, které prohlížeče ignorují.

V CSS existuje jen jeden jejich druh. Začínájí s /* a končí s */, přičemž mezi tím může být cokoliv.

        p {
            color: red; /* červená */
        }

Mohou být víceřádkové.

        p {
            color: red;
        }
/* pravidlo výše specifikuje...
...červenou barvu u odstavců. */

Shrnutí

  • V CSS se používá jen jedna struktura kódu - pravidla
  • Pravidla se skládají ze dvou částí:
    • Selektor: odkazuje na element, jež bude stylován
    • Deklarace: obaleny složenými závorkami {}. Skládá se ze dvou částí - vlastnosti a hodnoty, přičemž obojí je odděleno dvojtečkou :. Deklarace jsou zakončeny středníkem ;.
  • Na řádkování nezáleží
  • Níže položená pravidla mají vyšší prioritu než ta, která jsou položena výše
  • Komentáře začínají s /* a končí s */
