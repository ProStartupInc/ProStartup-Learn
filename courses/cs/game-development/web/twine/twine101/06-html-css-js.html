6. HTML, CSS, JS

Hlavní strana > Videohry > Webové hry > Twine > Twine 101> Lekce 6

HTML, CSS a JavaScript. Tři absolutně základní technologie, co se týče vývoje webových stránek a aplikací. Tyto technologie ale hrají významnou roli i ve vývoji Twine her - ty jsou přece také webové aplikace. V této lekci si proto ukážeme, jak je použít.

Poznámka: Pokud nechcete tyto tři technologie používat nebo je neumíte, můžete tuto lekci přeskočit.

HTML

HTML je značkovací jazyk používaný k vytváření webových stránek a jejich obsahu.

Hry vytvořené v Twine nejsou ničím jiným než obyčejnými .html soubory (ačkoliv obsahují rozsáhlý JavaScript, který do nich vložil vybraný příběhový formát). To znamená, že HTML v nich můžete používat tak, jak se vám zachce - můžete například každé pasáži vytvořit vlastní rozložení (v kombinaci s CSS) nebo jednoduše vkládat multimédia.

CSS

Okno, kde lze editovat CSS příběhu.

CSS je většinou používán v kombinaci s HTML. Stará se o vzhled a (ve většině případů) rozložení HTML dokumentů. A k přesně stejnému účelu ho lze použít v Twine.

U každého příběhového formátu se používání CSS liší, proto je nutné si vše v příslušných dokumentacích zjistit. Většinou ho lze editovat v okně, které se otevře po kliknutí na položku "Stylesheet" v kategorii "Příběh".

Poznámka: Příklad níže používá Harlowe.

JavaScript

Okno, kde lze editovat JavaScript příběhu.

JavaScript (zkráceně JS) je jazyk, který webovým stránkám přidává interaktivitu. Hry vytvořené v Twine sice nějakou mají, ale JS to může posunout za hranice "obyčejného" klikání na odkazy.

Implementace JS do příběhu se opět liší formát od formátu, takže bude zase nutné se podívat na příslušné dokumentace - zatímco některé formáty podporují pouze obyčejný JS, mnohé jeho funkce nahrazují s vlastními tzv. makry. Všeobecně lze JS editovat kliknutím na položku "JavaScript", které se nachází v horním panelu hned vedle tlačítka "Stylesheet".

TwineScript

TwineScript je skriptovací jazyk založen na JavaScriptu, jež je specificky určen pro vývoj v Twine. Přestože není vázán k nějakému specifickému příběhovému formátu, každý ho implementuje jinak. Budeme ho probírat až v kurzech o jednotlivých příběhových formátech.
